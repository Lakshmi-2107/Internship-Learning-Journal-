# 📊 Dashboards

## 📌 Overview
Dashboards combine multiple charts into one interactive interface for monitoring and decision-making.

---

## 🧩 1. Core Concepts

### 🔹 Components
- **Data Sources** → Connected datasets
- **Dimensions** → Categories (e.g., Country)
- **Metrics** → Numerical values

---

## 🏗️ 2. Building a Dashboard

### 🔹 Step 1: Data Preparation
- Single header row
- No merged cells
- Consistent data types

---

### 🔹 Step 2: Visual Elements

- **Tables** → Detailed view
- **Scorecards** → KPIs
- **Bar/Time Series Charts** → Trends
- **Heatmaps** → Comparisons
- **Geo Maps** → Location insights

---

### 🔹 Step 3: Interactivity

- **Date Filters**
- **Dropdown Filters**
- **Chart-as-Filter**

---

## 🎨 3. Dashboard Design

- Use consistent colors
- Organize layout clearly
- Create sections for readability

---

## ⚡ 4. Advanced Dashboarding

### 🔹 googleVis (R)
- Integrates R with Google Charts

### 🔹 Visualizations
- Sankey diagrams
- Motion charts

---

## 🎬 5. Animation in Dashboards (Flourish)

### 🔹 Purpose
- Reduce cognitive load
- Improve storytelling

### 🔹 Concepts
- **Object Constancy**
- **Drawing vs Morphing**

### 🔹 Examples
- Bar Chart Race
- Line Chart Race
- Animated Scatter Plot

---

## ✅ Key Takeaway
Dashboards:
- Combine insights  
- Enable interactivity  
- Support real-time decisions
